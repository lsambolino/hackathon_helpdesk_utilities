# CLAUDE.md — Conventions for this repo

This file teaches Claude Code (and any agent or contributor) how we work in
this repository. Keep it short. Update it when a convention changes.

## Project shape

- **Backend**: FastAPI single app (`app/main.py`), Python 3.11+.
- **DB**: SQLite, file at `data/helpdesk.db`, regenerated by `python -m app.seed`.
  Schema lives in `app/schema.sql`. Never hand-edit the DB file.
- **Agents**: Claude Agent SDK (Python). Coordinator in `app/agents/coordinator.py`,
  specialists in `app/agents/specialists/`, shared MCP tools in `app/agents/tools.py`.
- **Frontend**: Plain HTML + vanilla JS in `app/frontend/`. No build step. No npm.
- **Evals**: `evals/runner.py` over `evals/eval_set.json` (+ `evals/adversarial.json`).

## Conventions

- **One module = one responsibility.** Tools that read the DB live in `tools.py`,
  not inside specialist files. Specialists only choose *which* tools to call.
- **All tools return `{"content": [...], "is_error": bool}`** as the SDK expects.
  On failure return `is_error: True` with a human-readable message — don't raise.
- **Confidence + escalation are first-class.** Every coordinator decision writes
  `confidence` and (if escalated) `escalation_reason` to the `tickets` table.
- **Audit everything.** Each tool that mutates state must call `db.audit(...)`
  with the actor (`coordinator`, `billing_agent`, etc.) and a short detail string.
- **No real PII.** Synthetic Italian names only. Never paste real customer data.
- **Italian copy** in customer-facing strings (KB, ticket bodies, chatbot replies).
  Code, comments, schema in English.

## Running things

```bash
python -m app.seed                    # reset DB
uvicorn app.main:app --reload         # API + UI on :8080
python -m evals.runner                # eval harness, writes evals/report.json
python -m evals.runner --live         # uses the SDK against the configured provider
python -m scripts.probe_bedrock       # one-shot Bedrock health check
```

## Provider configuration

The SDK is provider-agnostic. Pick exactly one:

- **Anthropic API:** `export ANTHROPIC_API_KEY=sk-ant-...`
- **AWS Bedrock:** `export CLAUDE_CODE_USE_BEDROCK=1`, `AWS_REGION=...`, `AWS_PROFILE=...`

The model name is selected by `app/agents/coordinator.py::_model_for(role)` —
do NOT hard-code model strings elsewhere. Override at runtime with `HELPDESK_MODEL`.

## Permissions / human-in-the-loop

The Coordinator's `can_use_tool` callback lives in `app/agents/coordinator.py`.
It enforces these escalation rules — change them there, not in prompts:

- `confidence < 0.75` → escalate
- `invoice.amount_eur > 500` → escalate
- `customer.vulnerable == 1` → escalate
- adversarial signal detected (prompt-injection patterns) → escalate

## Don't do

- Don't add a frontend framework (React/Vue/Svelte). The plain HTML is intentional — judges run this in 60 seconds.
- Don't hit external APIs. Everything is local SQLite + the Anthropic API.
- Don't expand the schema without updating `seed.py` in the same commit.
- Don't commit `data/helpdesk.db` (it's gitignored; regenerated from `seed.py`).
